LAB_NUMBER: 01
LAB_TITLE: Configure Azure Blob Storage Access Tiers
ESTIMATED_DURATION: 45 Minutes

DESCRIPTION:
Learners will work inside a pre-provisioned Azure environment to explore Azure Blob
Storage access tiers. They will navigate to an existing Storage Account, create a
blob container, upload a sample text file as a blob, manually change its access tier
from Hot to Archive using the Azure Portal, and verify the tier change via blob
properties. The lab should reinforce the concept of cost optimisation through tiering
and explain what Archive means in practical terms (offline, rehydration required).

TASKS:
  TASK_1: Explore the Storage Account and Create a Blob Container
  TASK_2: Upload a Sample Blob to the Container
  TASK_3: Change the Blob Access Tier to Archive and Verify

AZURE_SERVICES:
  - Azure Storage Account (General Purpose v2, pre-provisioned)
  - Azure Blob Storage (Block Blobs only — access tiers not supported on Append or Page blobs)

PREREQUISITES:
  - Completion of the Getting Started page
  - Azure Portal open and signed in with lab credentials

NOTES:
  - Storage account is already provisioned — learners navigate to it, they do NOT create it
  - Use inject key for DeploymentID in the storage account name wherever it appears
  - Use inject key for Region wherever region is referenced
  - Archive tier is only supported on Block Blobs — mention this as a Note in the relevant step
  - After setting to Archive, the blob becomes offline — mention rehydration concept briefly
  - Include a Note that early deletion charges apply if blob is removed before 180 days
  - Add placeholder screenshot references after each major portal action
  - AI must reason the full detailed portal navigation steps on its own from the task descriptions
