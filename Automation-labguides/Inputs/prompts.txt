LAB_TITLE: Configure Azure Blob Storage Access Tiers
OVERALL_ESTIMATED_DURATION: 1 hour 30 minutes

DESCRIPTION:
This lab introduces learners to Azure Blob Storage and the concept of access tiers.
Learners will explore how Azure Storage organises data using containers and blobs,
and how different access tiers — Hot, Cool, Cold, and Archive — affect cost and
retrieval performance. By the end of this lab, learners will have hands-on experience
navigating the Azure portal to manage a storage account, uploading blobs, and
modifying access tier settings to optimise storage costs for different data lifecycle scenarios.

OBJECTIVES:
  - Explore the Azure Storage Account and understand its components
  - Upload a blob to a container using the Azure Portal
  - Change the access tier of a blob from Hot to Archive
  - Verify the tier change and understand its implications on retrieval

PREREQUISITES:
  - Basic familiarity with the Azure Portal navigation
  - Understanding of cloud storage concepts (optional but helpful)
  - No prior Azure Storage experience required
  - An active Azure subscription and lab environment credentials are pre-provisioned

AZURE_SERVICES:
  - Azure Storage Account: The core resource that hosts all storage services including blobs, files, queues, and tables
  - Azure Blob Storage: Object storage for unstructured data such as documents, images, videos, and logs
  - Azure Portal: The web-based management interface used throughout this lab to interact with Azure resources

ARCHITECTURE_DESCRIPTION:
The lab environment pre-provisions a single Azure Storage Account within a dedicated
resource group. Learners interact with the storage account via the Azure Portal to
manage blob containers and individual blobs. The access tier settings are configured
at the blob level, demonstrating how Azure moves data between Hot, Cool, and Archive
tiers within the same storage account without requiring data migration.

COMPONENTS:
  - Azure Resource Group: A logical container that holds all lab resources, making management and cleanup straightforward
  - Azure Storage Account: The parent resource provisioned in the lab environment — learners do not create this, it already exists
  - Blob Container: A logical grouping inside the storage account where blobs (files) are stored and organised
  - Blob (Object): The individual file uploaded during the lab — a sample text file used to demonstrate tier changes
  - Access Tier Setting: The property on each blob (Hot, Cool, Cold, Archive) that controls retrieval speed and storage cost
